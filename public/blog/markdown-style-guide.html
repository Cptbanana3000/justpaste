A Simple Markdown Style Guide

Consistency makes notes easier to read and maintain. Adopt a few simple rules and stick to them.

Headings

Use # for the title, then start sections at ##. Avoid skipping levels.

Lists

Use dashes for unordered lists and numbers for steps. Keep items parallel in structure.

Code

Include language tags and keep lines under ~80 characters when possible.

Links

Use descriptive text, not bare URLs, and place references at the end if needed.

Small standards pay big dividends over time.
