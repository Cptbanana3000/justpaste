A Beginner's Guide to Using Markdown for Notes

Markdown is a lightweight syntax that turns plain text into formatted content. It’s perfect for quick notes, checklists, and documentation—especially in tools like Flingnote. Here’s a fast introduction.

Headings

Use # for headings. The number of # symbols indicates the level:

# Title
## Section
### Subsection

Emphasis

*italics* or _italics_
**bold** or __bold__

Lists

- Item one
- Item two
- Item three

Checklists

- [ ] Task pending
- [x] Task done

Links

[Flingnote](https://flingnote.click)

Code

Inline code: `npm start`

Fenced code blocks:
```js
console.log('Hello Markdown');
```

Why Markdown?

  • It’s easy to write and read in plain text.
  • It’s fast—no heavy editors needed.
  • It’s portable across apps and platforms.

Tips for Better Markdown Notes

  • Use headings to organize sections.
  • Keep lists short and focused.
  • Use code blocks for commands or snippets.
  • Preview before sharing for clarity.

With a few basics, your notes will be clean, readable, and easy to share—perfect for Flingnote.
